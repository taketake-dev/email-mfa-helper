# 要件

## 目的

RD講義資料ページでID・パスワード入力後に表示されるメール二段階認証を、大学Outlook Webに届く6桁コードの取得とMoodle入力欄への設定で補助する。

## 想定利用者

- RD講義資料ページ、Chrome、大学Outlookメールを利用する学生。
- 初回だけ、利用者本人がChrome内でOutlook Webにログインする。
- MoodleのID・パスワード、Outlookのパスワード、送信操作は自動化しない。

## 利用者の流れ

1. 利用者がMoodleでID・パスワードを入力する。
2. 初回は、受信トレイ一覧を一時確認する処理を説明し、利用者が同意する。
3. 同意済みなら、二段階認証画面の検出を起点に拡張機能がOutlook Webの専用タブを前面で開く。
4. Outlook Webの受信トレイ一覧が表示された後、拡張機能は最大10秒、送信元・件名・認証開始時刻に基づいて対象メールを確認する。
5. 対象メールの一覧行に表示されるテキスト（本文プレビューを含む場合がある）から6桁コードを抽出し、Moodleの入力欄へ設定する。
6. 取得後はMoodleタブを前面に戻してから、拡張機能が開いたOutlookタブを閉じる。10秒以内に見つからない場合はOutlookタブを残したままMoodleタブへ戻り、利用者にOutlook Webでの確認を案内する。拡張機能は送信ボタンを操作せず、コード入力後のMoodle側の自動送信に任せる。

## 機能要件

- Moodleの対象二段階認証画面でのみ開始する。
- 初回の明示同意前と、同意を取り消した後はOutlook Webを開かず、認証メールを自動確認しない。
- 対象メールは送信元`noreply@lms.rd.dendai.ac.jp`、件名`あなたの認証コードです。`、受信日時で絞り込む。
- 受信日時は分単位表示のため、Moodleの二段階認証画面を最初に検出した時刻を分単位に切り捨て、その時刻以降を候補にする。初回同意やOutlookの表示が遅れても、この基準は変えない。
- Outlook未ログイン時は利用者本人がログインする。受信トレイ一覧が表示されるまで自動検索の10秒は開始しない。時間切れなら、Moodle上に状態を表示して自動処理を終了する。

## 非機能要件

- MoodleとOutlook Web以外へのアクセス権限を要求しない。
- 認証コードはMoodleの入力欄への設定以外で表示しない。対象行の表示テキストは端末内だけで一時処理し、メール本文、認証用リンク、資格情報、メール履歴を保存・外部送信・第三者共有せず、開発者が閲覧することもない。
- 動作中だけ、拡張機能が開いたタブIDと認証開始時刻を`chrome.storage.session`に保持し、取得後または時間切れで削除する。
- 同意状態だけを`chrome.storage.local`に保持し、設定画面で取り消せる。
- 大学管理者の承認、Microsoft Graph API、メール転送、外部サーバーに依存しない。

## 対象外

- MoodleのID・パスワード入力、送信ボタン押下、ログイン完了の自動化。
- 「この1日用デバイスを信頼します。」のチェック状態の変更。
- 大学Outlook以外のメールサービスへの転送・連携。
- 大学アカウントの共有や二段階認証そのものの無効化。
- Chrome Web Storeへの最終提出、開発者登録、登録料支払い。
