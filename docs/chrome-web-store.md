# Chrome Web Store 公開準備

最終更新: 2026-07-21

この文書は、Email MFA Helper を Chrome Web Store で公開するための準備、審査上の注意点、公開手順をまとめる。現在の初期公開方針は、ストア検索に表示しない **Unlisted（限定公開）** とする。

## 公開範囲の方針

- **Private（テスター限定公開）**: 指定したテスターだけがインストールできる。URLを友人以外へ渡したくない事前試用用。
- **Unlisted（限定公開）**: 検索結果には表示されないが、ストアURLを知る人はインストールできる。友人への配布と大学用Xアカウントからの案内に使う。
- **Public（公開）**: 検索結果を含め、誰でも発見・インストールできる。大学・Moodle・Microsoftの権利や運用ルールに問題がないことを確認してから切り替える。

3つの公開範囲は、同じChrome Web Storeポリシー要件と審査プロセスの対象である。Privateから段階的に公開しても、審査を軽くする効果はない。初回申請はUnlistedとし、必要があれば後からPublicへ切り替える。

## 審査期間と通過率

- 公式の目安は、多くの拡張機能で数日、長い場合は数週間である。
- 2026年4月時点で、Chrome Web Storeは提出増加による審査遅延を案内している。申請後は1週間程度を基準にし、3週間を超えて保留なら開発者サポートへ問い合わせる。
- 公式の合格率・却下率の統計は公開されていない。そのため、通過率を数値で見積もったり、「ほぼ必ず通る」と判断したりはしない。
- 新規開発者、新規拡張機能、機微な権限、確認しづらいコード、大きな変更は確認が厚くなり、審査が長引く場合がある。

## 現在の実装の評価

通過を目指せる材料:

- 目的は「Moodleの二段階認証に必要な、Outlook Web上の認証コード入力の補助」に限定されている。
- 対象ホストは `lms.rd.dendai.ac.jp`、`outlook.office.com`、`outlook.cloud.microsoft` に限定されている。
- 外部サーバーへの送信、メール本文・認証コードの永続保存、パスワード入力・送信ボタンの自動操作を行わない。

注意が必要な点:

- Outlook Webの受信トレイから認証メールの情報と6桁コードを一時的に読むため、個人通信および認証情報を扱う。端末内だけの処理でも、ユーザーデータの開示対象である。
- 利用者が処理開始前に内容を理解し、明示同意する画面が必要である。
- 大学・Moodle・Microsoftが公式に提供または承認した拡張機能であるような表現をしない。大学名・ロゴなどの利用も、許可がない限り避ける。
- Chrome Web Storeの承認は、大学のMoodle運用ルールやMicrosoftなど第三者の利用規約への適合を保証しない。Publicへ切り替える前に別途確認する。

## 申請前チェックリスト

### 実装

- [x] 初回利用時に、次を明示して利用者の同意を取得する。
  - Outlook Webの受信トレイ一覧から、指定送信元・件名の認証メールおよび6桁コードを端末内で一時的に読むこと。
  - 取得したコードをMoodleのコード欄へ入力すること。
  - メール本文・パスワードを保存または外部送信しないこと。
- [x] 同意しない場合は、Outlook Webを開かず、メール情報を読まない。
- [x] 同意状態を設定画面から取り消せるようにする。
- [x] 権限を再点検し、各権限の利用目的を説明できるようにする。
  - `alarms`: 10秒の待機終了を管理する。
  - `storage`: 実行中セッションを保持する。
  - 専用タブの作成・前面化・終了には、対象サイトへのホスト権限を使用する。`tabs` 権限は要求しない。
  - ホスト権限: Moodle認証画面の検出と、Outlook受信トレイ一覧の対象メール確認に限定する。
- [ ] 未ログイン、メール未着、画面変更など、取得できない場合の案内を確認する。
- [ ] 実機で正常系・失敗系を再確認する。

### プライバシーと表記

- [x] 公開URL用のプライバシーポリシーを`site/privacy.html`に用意する。GitHub Pagesの有効化後に公開される。
- [x] プライバシーポリシーに、扱う情報、利用目的、外部送信・第三者提供の有無、保存期間、問い合わせ先を記載する。
- [x] Chrome Web StoreのPrivacy practices欄に入力する回答案を用意する。
- [x] 拡張機能内の同意文、ストア説明、プライバシーポリシー、実装の内容を一致させる。
- [x] ストア説明とX投稿に使う非公式表記を用意する。

### ストア掲載素材

- [x] `manifest.json` と配布ZIPに、16/32/48/128pxの拡張機能アイコンを追加する。
- [x] Email MFA Helper の実際の利用画面を示すスクリーンショットを3枚用意する。個人情報、メールアドレス、認証コードはマスクする。
- [x] ストアの説明文、言語、サポート先URLを用意する。カテゴリは提出時にダッシュボードで選ぶ。
- [x] 小さなプロモーション画像（440x280px）を用意する。
- [x] 誇張、無関係なキーワード、公式と誤認させる表現を含めない。

### 開発者アカウントと提出

- [ ] Chrome Web Store開発者登録を行い、開発者契約・ポリシーに同意し、登録料を支払う（本人操作）。
- [ ] 開発者アカウントで2段階認証を有効にする。
- [x] `manifest.json` の名前、バージョン、説明、権限を確認する。
- [x] リリース用ZIPには拡張機能のファイルだけを含め、認証コード、メール内容、ローカル設定、秘密情報を含めないことを確認した。
- [ ] GitHubリポジトリをPublic化し、GitHub Pagesの公開元をGitHub Actionsに設定する（本人操作）。
- [ ] 初回はUnlistedを選び、審査へ提出する（本人操作）。
