# Chrome Web Store 掲載文案

## 公開方針

初版は **Unlisted（限定公開）** とする。ストア検索には表示されないが、URLを知る人はインストールできる。大学、Moodle、Microsoftの公式ツールではない。

## 短い説明

Moodleのメール二段階認証で、Outlook Webに届く6桁コードの入力を補助します。

## 詳細説明

Email MFA Helperは、Moodleの二段階認証画面で、利用者の同意後にOutlook Webの受信トレイ一覧を確認し、対象認証メールの6桁コードをMoodleの入力欄へ設定するChrome拡張機能です。

- 対象メールは、送信元、件名、受信日時で絞り込みます。
- 対象行に表示されるテキストは本文プレビューを含む場合があり、端末内で一時処理して6桁コードを抽出します。
- 受信トレイ一覧を表示した後、最大10秒だけ自動確認します。
- 見つからない場合はMoodleへ戻り、Outlook Webを残して手動確認を案内します。
- パスワード入力、Moodleの送信ボタン操作、認証コードのMoodle入力欄以外への表示は行いません。6桁入力後はMoodle側の仕様により自動送信されます。
- 認証コードやメール情報を保存、外部送信、第三者共有せず、開発者その他の人が閲覧することもありません。

この拡張機能は、大学、Moodle、Microsoftとは提携・承認関係にない非公式ツールです。利用するMoodleとOutlookの運用ルールを確認したうえで、自己責任で利用してください。

## Privacy practices回答案

- **単一の目的**: Moodleのメール二段階認証で、利用者が同意したOutlook Webの認証コードをMoodle入力欄へ設定すること。
- **扱うデータ**: Outlook Webの受信トレイ一覧にある対象メールの送信元、件名、受信日時、本文プレビューを含む場合がある一覧行の表示テキスト、およびそこから抽出した6桁認証コード。
- **利用方法**: 対象メールの判定と、Moodleのコード欄への一時入力だけに使用する。
- **保存・共有**: 認証コードとメール情報は保存・第三者共有・外部送信せず、人による閲覧も行わない。同意状態だけを端末内に保存する。
- **権限の理由**:
  - `storage`: 同意状態と実行中セッションを端末内で管理する。
  - サイトアクセス: Moodleの二段階認証パスと、Outlook Webの`/mail/`配下だけで静的コンテンツスクリプトを実行する。`tabs`、`alarms`、明示的な`host_permissions`は要求しない。
- **限定的な利用**: Chrome Web Store User Data Policy（Limited Use要件を含む）に従い、上記の単一目的に必要な範囲でだけデータを端末内処理する。
