# Moodle画面仕様

この文書は、RD講義資料ページのログインと二段階認証について確認済みの画面上の挙動を記録する。Moodle側の変更が疑われる場合は、この内容と実画面を照合する。

## ログイン画面

- ユーザー名入力欄がある。
- パスワード入力欄がある。
- 利用者が両方を入力し、ログインボタンを押す。

拡張機能は、この画面の入力やログインボタンを操作しない。

## 二段階認証画面

- 6桁認証コードの入力欄がある。
- 「この1日用デバイスを信頼します。」は、既定でチェックされている。
- 6桁コードが入力された時点で、Moodle側が自動送信する。
- 不正な認証コードではエラーが表示され、試行回数は10回までである。

## メール

- 認証メールは、認証開始からおおむね2〜10秒で届く。
- 送信元は`noreply@lms.rd.dendai.ac.jp`である。
- 件名は`あなたの認証コードです。`である。
- Outlook Webの受信トレイ一覧では、送信元・件名・受信日時・6桁コードを確認できる。拡張機能はメールを開かずに一覧行を確認する。
- Outlook Webの受信日時は分単位で表示され、秒単位では判別できない。
- 認証コードを使わずに再度ログインや二段階認証を開始しても、10分以内は新しい認証メールが送られない。
- 未使用の認証コードは発行から10分間有効であり、その間は同じコードを使う。
- 認証コードを使った後、または10分を過ぎた後に二段階認証を開始すると、新しい認証コードが発行される。
- 一度成功してログインするとパスコードはもう使えない

認証コードの有効期間（10分）と、拡張機能が受信トレイ表示後に新着メールを自動検索する時間（10秒）は別の設定として扱う。
